# 🌐 CASCADE VPN

Каскадный VPN-менеджер: один CLI на российском сервере управляет всей системой обхода блокировок — разворачивает VLESS+Reality (опц. XHTTP), MTProto-прокси и iptables-relay на нескольких серверах сразу. Есть веб-панель для админа.

```
Клиент → РФ-сервер (мост, iptables DNAT) → Сервер(ы) выхода (Xray + mtg) → Интернет
```

Зашёл в терминал РФ-сервера, ввёл одну команду — всё установилось. Нужна вторая локация (США рядом с Финляндией) — добавил выход из меню; клиент получит профиль на каждую. Управлять можно из терминала или из браузера (веб-панель за техдоменом).

## Возможности

- **Единая точка управления** — команда `cascade` на РФ-сервере, управление выходами по SSH
- **VLESS+Reality** — маскировка под легитимный HTTPS (Xray, raw config + systemd)
- **Несколько серверов выхода** — добавляешь локации (Финляндия, Германия…), у каждого свой relay-порт на мосту; клиент получает профиль на каждую
- **Мульти-клиенты** — добавить/удалить из меню/панели, один uuid на всех выходах; отозвать без перенастройки остальных
- **Сплит-туннель в профиле** — РФ-сайты с РФ IP устройства, иностранное через каскад; готовый JSON для Happ/v2rayNG/Nekoray (cascade или direct)
- **Веб-панель** — браузерная админка за техдоменом (Caddy авто-TLS): клиенты, выходы, настройки, ссылки-приглашения на 24ч; `/` — безобидная заглушка, вход на `/boss`
- **Смена SNI / имя в ссылке** — без переустановки
- **XHTTP+CDN** (опц., нужен домен) — резервный канал через Cloudflare
- **MTProto** — Telegram-прокси с Fake-TLS (mtg)
- **iptables-relay** — РФ-сервер как «тупая труба», трафик не расшифровывается
- **Мониторинг** — каждые N минут, Telegram-уведомления, авто-рестарт упавших выходов
- **Ссылки + QR** в терминале, интерфейс на русском, тема лайм-на-чёрном

## Установка

На РФ-сервере (Ubuntu/Debian) под root:

```bash
curl -sSf https://raw.githubusercontent.com/78ers/cascade/main/install.sh | bash
```

После установки — команда `cascade`.

## Использование

```
╔══════════════════════════════════════╗
║          🌐 CASCADE VPN              ║
║  🇫🇮 Финляндия  ● работает           ║
╚══════════════════════════════════════╝

  1. 🔧 Установка и настройка
  2. 🛡️  Управление VPN (VLESS+Reality)
  3. ✈️  Управление MTProto
  4. 📊 Статус и диагностика
  5. ⚙️  Настройки
  6. 🌐 Веб-панель
  7. 🚪 Выход
```

Навигация: стрелки ↑↓ или цифры, Enter — выбор.

**Первый запуск** (пункт 1): спрашивает IP и SSH-доступ сервера выхода, опц. домен для XHTTP — ставит Xray и mtg на выход, настраивает relay на мосту, показывает ссылки и QR.

**Выходы:** «Управление VPN» → «Выходы» — добавить локацию (деплой Xray по SSH + relay-правило) или удалить. Клиенту профиль генерится на каждый выход (cascade — через мост; direct — напрямую к выходу, для админа).

**Веб-панель** (пункт 6): задаёшь техдомен + пароль — CLI пишет Caddyfile/systemd и поднимает панель. Открываешь `https://<домен>/boss`. Внутри: клиенты, выходы, настройки, кнопка «Поделиться» (ссылка-приглашение `/c/<токен>` на 24ч с профилями+QR). `/` показывает безобидную заглушку.

## Зависимости

Ставятся автоматически: `python3`, `questionary`, `requests`, `flask`, `waitress`, `qrencode`, `iptables-persistent`, `caddy` (для веб-панели). На серверах выхода: Xray-core и mtg скачиваются с официальных релизов.

## Где что лежит

| Путь | Назначение |
|---|---|
| `/opt/cascade` | сам пакет |
| `/usr/local/bin/cascade` | команда запуска (`cascade`, `--monitor`, `--panel`) |
| `/etc/cascade/config.json` | конфиг (выходы, клиенты, панель, настройки) |
| `/etc/cascade/panel_secret` | ключ подписи сессий панели (0600) |
| `/etc/caddy/Caddyfile` | конфиг Caddy для веб-панели (на мосту) |
| `cascade-panel.service` | веб-панель (waitress, на мосту) |
| `/usr/local/etc/xray/config.json` | Xray на сервере(ах) выхода |
| `cascade-mtg@<port>.service` | mtg на сервере выхода |
